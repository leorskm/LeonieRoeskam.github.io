Analysing Student Data

This project was part of an assignment in a module during my final year at university. Using Excel and Python, I performed linear regressions and identified outliers.

  • Excel
  • Python

This university project, completed during my final year, involved analysing a dataset generated from two surveys completed by students - one collecting basic personal information and the other assessing personality traits using the 16Personalities test. To compensate incomplete responses and inconsistencies, I used linear regression techniques in Excel to impute missing data and Python to detect outliers. The dataset included various metrics such as height, shoe size, distance to university, and personality traits such as extraversion versus introversion. I also merged the survey tables using Excel's VLOOKUP, although the low completion rate resulted in a smaller dataset, so I focused on exploring and explaining anomalies within the data from the first survey.

Analysing Netflix Data

The objective of this Python project is to perform basic statistical analysis and data visualization on a dataset containing information about shows and movies available on Netflix.

  • Python

This Python project focuses on conducting basic statistical analysis and data visualization on a dataset detailing Netflix's library of shows and movies. Using a dataset containing Netflix data, containing information on identifiers, content type, title, director, production country, addition date, release year, rating, and duration, the project explores content distribution, ratings, and trends over time through descriptive and inferential statistics, as well as linear regression. Key tasks include analyzing the prevalence of movies versus TV shows, examining rating distributions, and comparing content durations. Additionally, inferential statistics such as t-tests assess differences in release years between movies and TV shows, providing insights into content refresh rates. A linear regression model predicts release years based on content type and duration, evaluated by Mean Squared Error and R^2 scores.

Analysing Fuel Efficiency

The objective of this project was to develop a supervised machine learning model capable of predicting the combined fuel efficiency (measured in miles per gallon, MPG) of vehicles.

  • Python

This project developed a machine learning model for predicting vehicle fuel efficiency (MPG) using a dataset of over 38,000 entries detailing vehicle specifications, engine characteristics and emissions data. Extensive data cleaning involved removing columns with excessive missing data and imputing other gaps. Feature engineering focused on the selection and standardisation of attributes strongly correlated with MPG. A Gradient Boosting Machine (GBM) model was trained on an 80-20 split of the data and evaluated using R-squared and RMSE metrics. Results, including actual vs. predicted MPG comparisons and feature importance, were visualised to demonstrate the effectiveness of the model in predicting fuel efficiency.

Natural Language Processing

This university project aimed to understand and explore Natural Language Processing (NLP) techniques, specifically focusing on keyword extraction algorithms such as TF-IDF, to enhance customer service operations in the digital era.

  • Python

This university project focused on using natural language processing (NLP) techniques, specifically the Term Frequency-Inverse Document Frequency (TF-IDF) model, to improve customer service operations by automating responses and enhancing chatbot capabilities. Using Python and libraries such as sklearn, the project involved generating synthetic customer queries to simulate real-world interactions, which served as a dataset to test the NLP algorithm. The TF-IDF model has the ability to prioritise customer service responses by analysing the importance of words and phrases in these queries. In addition, word clouds were used to visually illustrate the most common terms before and after applying the TF-IDF algorithm.

Analysing Mental Health Data

The objective of this project is to demonstrate the ability to perform fundamental SQL operations, including creating and manipulating data within a relational database.

  • SQL

This project focuses on demonstrating basic SQL skills by creating and manipulating data within a relational database, focusing on responses to a mental health survey. There are two main tables in the project: SurveyResponses, which collects individual responses from a mental health survey, including age, gender and mental health treatment details; and CountryInfo, which provides demographic details such as region and population for each country represented in the survey. Key SQL tasks included creating and populating these tables, performing basic data manipulations such as updates and deletions, and querying the data to filter responses or calculate statistics such as average age. Advanced operations included using INNER JOINs to merge tables for richer analysis, and creating indices and views to improve query performance and simplify data access.

Analysing Environmental Data

The objective of this university project was to assess the environmental footprint of G20 nations by examining various sustainability metrics, using Power BI to create interactive visualizations.

This university project assessed the environmental footprint of the G20 nations by analysing sustainability metrics such as CO2 and methane emissions, energy use, and renewable energy use, using Power BI for data visualisation. Analysing Play Store Data

This project aimed to segment Google Play Store applications into meaningful clusters to understand patterns in app popularity, user engagement and monetisation strategies using unsupervised learning techniques.

  • R

This project focused on segmenting Google Play Store applications using unsupervised learning techniques, specifically K-Means clustering, to identify patterns in app popularity, user engagement, and monetisation strategies. The primary dataset included details such as app category, ratings, reviews, installs and more. Key tasks included extensive data cleaning, data format conversion and unit standardisation. Feature engineering included extracting temporal data and categorising price and installs for more nuanced analysis. The K-Means algorithm was used to group apps into clusters based on standardised features such as ratings and installs, which helped scale variables for better algorithm performance. Visualisations such as pie charts and scatter plots highlighted cluster distributions and relationships between app characteristics. Exploratory data analysis (EDA) further explored trends within each cluster to identify factors that influence app success.
